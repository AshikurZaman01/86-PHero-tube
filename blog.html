Var, Let, and Const

var

In JavaScript when you declare a variable using the keyword "var" its scope is limited to the function in which it is defined. If you declare a var variable outside of any function it becomes a variable. Can be accessed throughout the entire script. Another aspect to consider is hoisting. With var variables they are moved to the top of their containing function or global scope during the code execution. This means that you can use and access var variables before they are actually declared in your code.

For example:

var x = 10;
}
console.log(x);
}

let

Block Scope: Variables declared using the keyword "let" have block scope. This means that they can only be accessed within the block of code (enclosed in braces) where they are defined. This feature is especially handy when working with loops and conditional statements. No Hoisting; Unlike variables declared with "var" let variables are not hoisted to the top of their containing block. As a result you cannot access them before they are declared.

For example:
if (true) {
let y = 20;
}
console.log(y)

conts

Block Scope:Similar, to the 'let' keyword, 'const' variables are also limited, to the block where they are defined. They cannot be accessed outside of that block. Constants The 'const' keyword is used to declare constants, which means their values cannot be changed once they have been initialized. However even though the content of objects and arrays declared with 'const' can be modified the variable itself cannot be reassigned.

For example:
const PI = 3.14;
PI = 3.14159;

const myArray = [1, 2 3];
myArray.push(4);
myArray = [4, 5 6];

In summary, the scope of 'var' is function scoped or global if declared outside functions 'let' and 'const' are block scoped and const additionally enforces immutability for the variable binding itself it is generally recommended to use let or const over var to avoid unexpected scoping issues and to choose between let and const based on whether the variable s value should remain constant or can be changed

Rest API

Rest API

The Representational State Transfer Application Programming Interface (REST API) is a collection of architectural principles and limitations for creating and interacting with web services. It is a method of developing networked apps that use the HTTP protocol. REST is not a technology in and of itself, but rather a design approach for networked applications with specified properties. REST's fundamental principles and notions are as follows:

One of the essential concepts of REST is that each request from a client to a server must include all of the information required to comprehend and perform the request. In other words, the server should not keep track of the client's status between queries. Each request should stand on its own.

Resources: The key abstractions in REST are resources, which are commonly identified by URLs (Uniform Resource Locators). Resources can represent any entity, such as data objects, services, or real-world entities, and are modified using conventional HTTP protocols.

HTTP Methods: RESTful APIs employ HTTP methods (verbs) to accomplish resource actions. The following are the most frequent HTTP methods used in REST:

GET: Get information from the server.
POST: Create a new server resource.
PUT: Update an existing resource or create one if one does not already exist.
DELETE: This command removes a resource from the server.

REST APIs have a consistent and uniform interface, which means that the same set of HTTP methods and protocols are utilized across the API. This consistency simplifies both the API's architecture and interaction with it.

RESTful communication between client and server is stateless, which means that each request from the client to the server must include all necessary information. Client state is not saved by the server between queries. REST resources are provided in a format that both the client and server can understand. JSON and XML are common representation forms, however other formats can be utilized as well.

Layered System: REST allows for a layered architecture, where various components (e.g., proxies, load balancers, caches) can be added to improve system scalability and security without affecting the overall architecture.

REST APIs are widely used for building web services because of their simplicity and compatibility with HTTP, which is the backbone of the World Wide Web. They are commonly used for exposing data and services over the internet and are the basis for many web applications and mobile app backends. RESTful APIs have become a standard for building web services due to their ease of use, scalability, and wide support in programming languages and frameworks.
